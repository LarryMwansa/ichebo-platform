The Market Problem

Built for a reality that existing software was never designed to address.

"The problem is not that faith-based organisations lack the will to modernise. It is that no platform has been built that genuinely fits how they are structured and how they actually operate."

Existing church management software is designed for a Western, congregational model: a single pastor, a single congregation, a weekly service, and a membership database. It does not account for apostolic networks, multi-tiered denominational hierarchies, community-based discipleship systems, or the relational complexity of networked ministry organisations.

Ichebo does not compete with existing church management software. It serves the organisational reality that existing software was never designed to address.

Platform Architecture

The KGS five-level structure — made digital.

The Ichebo Platform mirrors the Kingdom Governance System's organisational architecture. Every tenant tier has defined permissions, data visibility, and governance authority. This is not a flat database — it is a hierarchical governance system expressed in code.

Architectural Decisions
account_tree
Materialised Path Tenant Hierarchy
Efficient querying across unlimited hierarchy depth without recursive joins. Supports organisations from a single local node to a global network of hundreds.
table_rows
Single Records Table with Discriminator
All institutional records — people, documents, communities, governance items — in a unified architecture. Structural consistency across the full diversity of record types.
security
Three-Dimensional Permission Matrix
Every data access, every governance action, every record modification is subject to permission logic aligned with KGS governance authority. Role × level × organisational scope.
auto_awesome
Paraclete Orchestration Layer
A standalone service managing cross-app workflows, event routing, and service coordination. Separation of concerns ensures platform complexity scales without degrading performance.

The Network Multiplier Effect

One network. Fifty
operational units.

The Ichebo Platform grows through network multiplication, not individual adoption. When a network-level organisation adopts the platform, every Sceptre Community and Church Collective within its governance structure becomes a potential active tenant.

A single apostolic network with 50 communities is not one customer — it is 50 operational units within a shared governance environment, each with its own data, governance, and accountability. Without the Church Node, multi-tenancy is only a software feature. With it, tenancy becomes governance architecture.
