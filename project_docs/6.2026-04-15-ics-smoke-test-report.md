## Known Issues

### 1. Governance API Not Yet Wired (Expected)
- `GET /api/governance/records/` returns 404
- Governance template views exist and are accessible
- API endpoints for governance will be implemented in Phase 7+

### 2. Activity `completed_at` Field (Minor)
- Activity marked as completed, but `completed_at` field is null
- Expected behavior: should be set to current timestamp on completion
- **Recommendation:** Add signal handler in `activity/signals.py`:
  ```python
  @receiver(pre_save, sender=Activity)
  def set_completed_at(sender, instance, **kwargs):
      if instance.status == 'completed' and not instance.completed_at:
          instance.completed_at = timezone.now()
  ```

### 3. Auto-Certification Logic
- User completed a programme activity (100% progress)
- Expected: Certification should be created
- Actual: Certification creation requires full programme completion in Learn app
- **Status:** Working as designed (enrolment ≠ Learn app programme)

## Recommendations for VPS Deployment

1. **Run migrations on fresh database:**
   ```bash
   python manage.py migrate --run-syncdb
   ```

2. **Create superuser for admin access:**
   ```bash
   python manage.py createsuperuser
   ```

3. **Collect static files:**
   ```bash
   python manage.py collectstatic --noinput
   ```

4. **Start Gunicorn with production settings:**
   ```bash
   DJANGO_SETTINGS_MODULE=ics_project.settings.production \
   gunicorn ics_project.wsgi:application -c gunicorn.conf.py
   ```

5. **Configure Nginx:**
   - Serve static files from `/var/cache/ics/staticfiles/`
   - Proxy `/api/` and `/` to Gunicorn (127.0.0.1:8000)
   - Enable SSL via certbot

6. **Set up systemd service:**
   - Create `/etc/systemd/system/ics-gunicorn.service`
   - Enable auto-restart on reboot
